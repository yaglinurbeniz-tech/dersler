Artık klavye değil, sesin konuşacak

Çeviri kontrolünü bitirdin — bot İngilizce cevabını doğru yargılıyor. Şimdi son parçayı takıyoruz: cevabı mikrofondan alacağız. Ve bu sefer kodu temiz yazacağız.

🎯 Bugünkü hedef: Türkçe cümle ekranda çıksın → sen İngilizce çevirisini söyle → program sesini kaydetsin, yazıya çevirsin, ve zaten yazdığın kontrol beynine versin.
🎤 Mikrofon→ kaydet()→ dinle()→ temizle()→ kontrol_et()→ puan

Kırmızılar bugün yazacakların. Yeşiller geçen ders yazdıkların — hiç değişmeyecekler. Çünkü onlar cevabın nereden geldiğini bilmiyor, sadece bir metin alıp karar veriyorlar. İşte iyi kod tam olarak budur.

Kütüphaneleri kur ve içeri al

+20 XP

Terminale şunu yaz (Jarvis'te speech_recognition'ı zaten kullanmıştın — tanıdık gelecek):

pip install sounddevice scipy SpeechRecognition

Sonra dosyanın en üstüne import'ları ekle:

import sounddevice as sd          # mikrofondan ses kaydeder
import scipy.io.wavfile as wav    # kaydı .wav dosyası yapar
import speech_recognition as sr  # sesi yazıya çevirir

recognizer = sr.Recognizer()
SAMPLE_RATE = 44100   # saniyede kaç ses örneği alınacak
DURATION = 4          # kaç saniye kayıt yapılacak
Neden 3 kütüphane? Her biri tek bir iş yapıyor: sounddevice mikrofonu dinler, scipy o sesi dosyaya yazar, speech_recognition dosyayı okuyup metne çevirir.

SAMPLE_RATE ve DURATION neden BÜYÜK harf? Python'da "bu değer değişmeyecek, ayardır" demenin yolu. Kod okuyan herkes bunu anlar — temiz kodun ilk kuralı.

kaydet() — mikrofonu aç, dosyaya yaz

+30 XP

Tek işi olan bir fonksiyon: belirtilen saniye kadar kaydet, wav dosyası olarak kaydet, dosya adını döndür.

def kaydet(dosya_adi="cevap.wav"):
    print("🎤 Konuş... (%d saniye)" % DURATION)

    kayit = sd.rec(
        int(DURATION * SAMPLE_RATE),
        samplerate=SAMPLE_RATE,
        channels=1,
        dtype="int16"
    )
    sd.wait()              # kayıt bitene kadar bekle

    # GÖREV: kaydı dosyaya yaz -> wav.write(dosya_adi, SAMPLE_RATE, kayit)
    ...
    # GÖREV: dosya_adi'ni return et
    ...
int(DURATION * SAMPLE_RATE) → "4 saniye × 44100 örnek = kaç örnek alacağım" hesabı. sd.wait() olmazsa program kayıt bitmeden devam eder ve elinde boş ses kalır. channels=1 tek mikrofon (mono) demek. Fonksiyon dosya adını döndürsün ki bir sonraki fonksiyon o dosyayı bulabilsin.

dinle() — sesi yazıya çevir (ve çökme!)

+35 XP

Kaydedilen dosyayı okuyup metne çevir. Ama internet kesilebilir, sen sessiz kalabilirsin — program çökmemeli. Bu yüzden try/except şart.

def dinle(dosya_adi):
    with sr.AudioFile(dosya_adi) as kaynak:
        ses = recognizer.record(kaynak)

    try:
        metin = recognizer.recognize_google(ses, language="en-US")
        print("Duyduğum:", metin)
        # GÖREV: metni return et
        ...
    except sr.UnknownValueError:
        print("Seni anlayamadım 🤔")
        return ""
    except sr.RequestError:
        # GÖREV: "İnternet/servis sorunu" yazdır ve "" döndür
        ...
Neden language="en-US"? Jarvis'te "tr-TR" yazmıştın çünkü Türkçe konuşuyordun. Burada İngilizce çeviri söyleyeceksin, o yüzden İngilizce.

İki farklı except neden? İkisi farklı dert: UnknownValueError = "ses vardı ama anlamadım" (sessiz kaldın), RequestError = "servise ulaşamadım" (internet yok). Ayrı ayrı yakalarsan oyuncuya doğru mesajı verirsin.

Neden boş string "" döndürüyoruz? Jarvis'te de öyle yapmıştın! Böylece çağıran taraf "boş geldi, demek ki anlaşılmadı" diyebilir.
Bağlama — asıl an

cevap_al() — input()'un yerine geç

+30 XP

İşte dersin can alıcı kısmı. Oyun döngüsündeki input(...) satırını silip yerine tek bir fonksiyon çağrısı koyacaksın. Kontrol beynine dokunmuyoruz.

def cevap_al():
    # GÖREV: önce kaydet(), dönen dosya adını dinle()'ye ver, sonucu return et
    dosya = ...
    return ...


# --- oyun döngüsünde SADECE bu satır değişiyor ---
# ESKİ:  cevap = input("İngilizce çevirisi: ")
# YENİ:
cevap = cevap_al()

# Aşağısı GEÇEN DERSTEN AYNI — hiç dokunma:
dogru_mu = False
for k in kabul_edilenler:
    if temizle(cevap) == temizle(k):
        dogru_mu = True
cevap_al() iki satır: dosya = kaydet() ve return dinle(dosya).

Buradaki büyük fikir: temizle() ve kontrol döngüsü, cevabın klavyeden mi mikrofondan mı geldiğini bilmiyor — sadece bir metin alıyor. Bu yüzden değiştirmene gerek yok. Fonksiyonları böyle ayırınca bir parçayı sökülüp yenisini takabiliyorsun. Buna "tek sorumluluk" denir ve profesyonel kodun temelidir.
